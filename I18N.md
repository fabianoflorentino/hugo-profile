# Internationalization (i18n) Support

The Hugo Profile theme now supports internationalization, allowing you to create multilingual websites.

## Overview

The theme uses Hugo's built-in i18n functionality to translate interface strings. Translation files are located in the `i18n/` directory.

## Available Languages

The theme currently includes translation files for:

- **English** (`en.toml`) - Default language
- **Spanish** (`es.toml`)
- **French** (`fr.toml`)
- **German** (`de.toml`)
- **Portuguese** (`pt.toml`)

You can easily add more languages by creating new translation files in the `i18n/` directory.

## Setting Up a Multilingual Site

### 1. Basic Configuration

To enable multilingual support, update your `hugo.yaml` configuration file:

```yaml
defaultContentLanguage: en
languageCode: en-us

languages:
  en:
    languageName: "English"
    weight: 1
    title: "Hugo Profile"
  es:
    languageName: "Español"
    weight: 2
    title: "Perfil Hugo"
  fr:
    languageName: "Français"
    weight: 3
    title: "Profil Hugo"
```

### 2. Language-Specific Content

Create language-specific content directories:

```
content/
├── en/
│   └── blogs/
│       └── my-post.md
├── es/
│   └── blogs/
│       └── my-post.md
└── fr/
    └── blogs/
        └── my-post.md
```

### 3. Example Configuration

See `exampleSite/hugo-multilingual.yaml` for a complete multilingual configuration example.

## Customizing Translations

### Using Existing Translation Files

Edit the translation files in the `i18n/` directory:

```toml
# i18n/en.toml
[about]
other = "About Me"

[experience]
other = "Work Experience"
```

### Adding a New Language

1. Create a new translation file in the `i18n/` directory (e.g., `i18n/de.toml` for German)
2. Copy the structure from `i18n/en.toml`
3. Translate all strings to the target language
4. Add the language to your `hugo.yaml` configuration

Example German translation file (`i18n/de.toml`):

```toml
[about]
other = "Über mich"

[experience]
other = "Erfahrung"

[education]
other = "Bildung"
```

## Overriding Default Text

You can still override default text using the configuration parameters. The theme will use them in this priority order:

1. Configuration parameter (e.g., `params.about.title`)
2. i18n translation
3. Built-in default

Example:

```yaml
params:
  about:
    enable: true
    title: "About Me"  # This will override the i18n translation
```

## Backwards Compatibility

The theme maintains backwards compatibility with sites that don't use i18n. If no i18n files are present or no translation is found, the theme will fall back to:

1. The `params.terms.*` configuration values (if set)
2. Built-in English defaults

## Testing Your Multilingual Site

Build and test your site:

```bash
hugo server
```

Then navigate to different language versions:
- English: `http://localhost:1313/en/`
- Spanish: `http://localhost:1313/es/`
- French: `http://localhost:1313/fr/`

## Language Switcher

To add a language switcher to your site, you can customize the navigation menu. Add this to your templates or create a custom partial:

```html
{{ if .IsTranslated }}
  {{ range .Translations }}
    <a href="{{ .Permalink }}">{{ .Language.LanguageName }}</a>
  {{ end }}
{{ end }}
```
